# Release Notes - v0.0.1

## Release Purpose
First field-validation release for offline GPS-to-radio phrase generation.

## Included Capability
- Offline PWA shell with service worker cache management.
- Resort Pack schema validation and semantic validation.
- IndexedDB persistence for pack import/select/active state.
- Live GPS map with nearest spatial context.
- Deterministic geometry and semantics engine.
- Deterministic radio phrase composer with fallbacks.
- Generate + copy phrase action flow.
- Acceptance tests mapped to roadmap criteria.

## Explicitly Not Included
- Dispatch integration.
- Shared patrol state/tracking.
- Sweep workflows.
- Incident record management.

## Verification Summary
- `npm run check` passes.
- Acceptance suite covers:
  - offline shell navigation fallback,
  - phrase generation under 2 seconds,
  - run and tower context inclusion,
  - copy action behavior.

## Tagging Procedure
Run from clean `main` after merge:

```bash
git checkout main
git pull --ff-only
git tag -a v0.0.1 -m "Patrol Toolkit v0.0.1"
git push origin v0.0.1
```

## Go/No-Go Gate
Go only if all are true:
- Field trial runbook executed.
- At least one successful live radio call with generated phrase.
- Patrol feedback documented.
- No blocker defects in offline behavior, run/tower context, or copy flow.
