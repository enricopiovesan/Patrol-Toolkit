# OSM Extractor Roadmap 1.3.0

## Objective
Deliver machine-readable pack validation output with actionable entity references for automation and CI diagnostics.

## Capabilities
- `validate-pack --json` output mode.
- Structured validation issues including:
  - schema path,
  - error keyword,
  - message,
  - entity reference (`run:<id>`, `lift:<id>`, `resort:<id>`) when available.
- Non-zero exit behavior preserved for invalid input.

## Acceptance Criteria
1. Valid pack returns machine-readable success output in JSON mode.
2. Invalid pack returns machine-readable issue list in JSON mode.
3. Validation issues include entity references when errors map to run/lift/resort scopes.
4. Tests cover structured issue generation and entity reference mapping.
