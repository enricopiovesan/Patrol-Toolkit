# OSM Extractor Roadmap 1.4.0

## Objective
Add explicit area configuration support so resort extraction can be scoped to a target geographic bounding box.

## Capabilities
- `ingest-osm` accepts `--bbox <minLon,minLat,maxLon,maxLat>`.
- Resort config supports `source.area.bbox`.
- Ingestion filters run/lift ways to features intersecting the configured bbox.
- Normalized output includes a deterministic warning summary of bbox filtering impact.

## Acceptance Criteria
1. Ingestion can filter extracted runs/lifts with a CLI bbox argument.
2. `extract-resort` honors bbox specified in config.
3. Invalid area config fails schema validation with actionable errors.
4. Tests cover bbox filtering behavior and config parsing.
