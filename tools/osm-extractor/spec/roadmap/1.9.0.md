# OSM Extractor Roadmap 1.9.0

## Objective
Make `--json` mode fully automation-safe by returning deterministic machine-readable error envelopes on command failures.

## Capabilities
- Standard JSON error payload for all CLI commands when `--json` is set.
- Error payload includes:
  - command name,
  - stable error code,
  - human-readable message,
  - optional structured details.
- `validate-pack --json` invalid input returns a structured error payload with issue details.
- Existing text error output remains unchanged when `--json` is omitted.

## Acceptance Criteria
1. Every command failure with `--json` returns valid JSON with stable top-level shape.
2. Validation failures expose structured issues for automation consumers.
3. Non-JSON mode behavior remains backward compatible.
4. Tests cover command-error formatter contracts.
